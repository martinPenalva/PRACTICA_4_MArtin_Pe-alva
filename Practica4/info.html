1. Contextualización

En los sistemas robóticos modernos, los sensores táctiles permiten que los robots perciban propiedades físicas del entorno, como presión, textura, forma y distribución de contacto.

Los sensores táctiles matriciales funcionan de manera similar a una imagen en escala de grises, donde cada celda representa el nivel de presión detectado en una posición específica.

En esta práctica se trabajará con un sensor táctil integrado en un gripper robótico. Los datos capturados han sido almacenados en formato JSON y contienen 50 capturas táctiles diferentes.

A partir de estas matrices, los estudiantes deberán aplicar técnicas de interpolación para aumentar la resolución hasta 128 × 128.

Conceptos fundamentales

  • Procesamiento de matrices
  • Interpolación numérica
  • Comunicación cliente-servidor
  • Procesamiento de señales táctiles
  • Visualización científica

2. Objetivos

Objetivo general

Desarrollar un sistema distribuido en C++ y Python capaz de procesar información táctil proveniente de un sensor robótico, aumentando su resolución mediante interpolación y generando representaciones visuales de la presión detectada.

Objetivos específicos

  • Leer y procesar archivos JSON en C++
  • Manipular matrices bidimensionales
  • Implementar interpolación bilineal
  • Desarrollar un cliente HTTP en C++
  • Implementar un servidor HTTP en Python
  • Generar imágenes utilizando Matplotlib

3. Descripción de la práctica

Se dispone de un conjunto de datos táctiles almacenados en un archivo JSON.

Tamaño de cada matriz: 16 × 16

Total de valores por captura: 256

Los estudiantes deberán:

  1. Leer las 50 capturas desde el archivo JSON
  2. Validar que todas las matrices tengan dimensiones 16 × 16
  3. Convertir cada matriz a 128 × 128
  4. Aplicar interpolación bilineal
  5. Enviar matrices al servidor Python
  6. Generar imágenes táctiles
  7. Guardar las imágenes generadas

4. Estructura del archivo JSON

{
  "captures": [
    {
      "id": 0,
      "matrix": [
        [0, 12, 14, 18],
        [0, 10, 15, 22],
        [0, 8, 18, 28]
      ]
    }
  ]
}

Cada captura contiene:

  • id: identificador de la captura
  • matrix: matriz táctil 16 × 16

6. Interpolación requerida

Escalado: 16 × 16 → 128 × 128

Factor de escala: 8x

La interpolación bilineal deberá implementarse manualmente. No está permitido utilizar OpenCV o SciPy para realizar directamente el escalado.

7. Comunicación cliente-servidor

El cliente en C++ deberá enviar matrices utilizando HTTP POST.

{
  "capture_id": 0,
  "width": 128,
  "height": 128,
  "data": [[...]]
}

8. Visualización en Python

plt.imshow(matrix, cmap="inferno")
plt.colorbar(label="Presion")
plt.title("Mapa tactil")

Las imágenes deberán guardarse automáticamente:

capture_0.png
capture_1.png
...
